powered by Somnia Agents

An on-chain firewall for autonomous AI wallets.

Agents that hold private keys are an unbounded liability — one prompt injection or hallucinated trade drains the wallet. AgentGuard puts a programmable checkpoint between an agent's intent and the chain: funds live in a vault, agents can propose but never execute, and every action is reviewed by consensus-verified Somnia Agents before a cent moves.

how it works
01Agent proposes — submits an action; it cannot execute directly.
02Vault holds — takes custody, pending review.
03Agents review — Parse-Website verifies context; Qwen3-30B LLM Inference judges intent vs policy.
04Verdict on-chain — returned by a validator subcommittee on strict-majority consensus, not a trusted oracle.
05Vault disposes — executes, timelocks, or blocks.
owner-defined policy
Spend caps · vault-ratio caps · 24h review timelock · target allowlist · blocked keywords. Caps are hard limits in the contract — even APPROVE can't exceed them.
every action gets a verdict — proven live on-chain
APPROVE Safe payment to an allowlisted recipient → executed. action #1 ✓
REVIEW Unrecognized payee → held behind a 24h timelock the owner can veto. action #5 ✓
BLOCK Suspicious drain → never executable; executeAction reverts on-chain. action #2 ✓
Each verdict is a real, explorer-verifiable transaction. Proof cards on the dapp read verdicts live from the contract — no wallet needed.
